# BOM 管理系統

面向 BOM（Bill of Materials，物料清單）的管理系統。提供物料主檔、BOM 結構（物料組成關係）維護，並支援 BOM 結構查詢、成本計算，以及以「替代方案」方式彈性套用替代料，計算替代後的成本。

---

## 功能特色

### 物料主檔管理
- 新增／查詢／刪除物料（編碼、名稱、單位、單價）
- 刪除物料時，若仍被任何 BOM 組成關係或替代方案規則引用，會擋下並列出引用位置，避免資料斷裂

### BOM 結構維護
- 定義「父物料的 BOM 裡包含子物料，數量多少」
- 一顆物料的組成只需定義一次，任何引用到它的地方（不同產品、不同上層組件）都會共用同一份，不需要重複建立
- 新增組成時會自動檢查父／子物料是否存在、是否會造成循環依賴（物料不可以直接或間接包含自己）
- 刪除一條組成關係只會移除該條父子關係，子物料自己的組成不受影響，仍可被其他地方引用

### BOM 查詢／成本計算
- 輸入根節點物料，查詢完整展開的樹狀結構（遞迴組合底下所有子物料）
- 計算 BOM 總成本，會考慮各層數量的累乘關係
- 查詢與計算成本時，可選擇性套用一個或多個「替代方案」，並針對每個方案影響到的主料輸入這次要替換的數量；同一顆主料允許被多個方案共同覆蓋（例如兩個供應商各補一半），但合計數量不可超過該主料的 BOM 需求量

### 替代方案管理
- 建立可命名的替代方案（例如「多供應商策略」），方案本身只描述「有哪些替代規則」，實際套用與否、套用多少數量由查詢當下決定
- 方案明細：定義「主料可以用哪個替代料取代、替代比例（1 顆主料對應幾顆替代料）、單價」
- 提供跨方案的替代料明細查詢，可用關鍵字搜尋所有方案裡「誰換誰」的規則，不需要先選定方案

## 技術棧

- **Spring Boot 3.3.5** ／ Java 17
- **MyBatis-Plus**：持久層 ORM，搭配原生 SQL（遞迴 CTE）查詢 BOM 展開結構
- **H2 Database**：檔案模式，資料存放於 `data/` 目錄
- **Caffeine**：BOM 結構／成本計算結果的本地快取，資料異動時自動清除
- **SpringDoc OpenAPI**：自動產生 Swagger UI 介面文件
- **Lombok**
- 前端：純 HTML + Vanilla JavaScript（無需額外建置工具）

## 快速開始

### 環境需求
- JDK 17 以上
- **不需要事先安裝 Maven**：專案內含 Maven Wrapper（`mvnw` / `mvnw.cmd`），會自動下載對應版本的 Maven，第一次執行需要能連線下載

### 啟動專案

Windows：

```bat
mvnw.cmd spring-boot:run
```

或直接執行專案根目錄下的 `run.bat`。

Linux／Mac：

```bash
./mvnw spring-boot:run
```

啟動後可透過以下網址存取：

| 項目 | 網址 |
|---|---|
| 前端操作介面 | http://localhost:8095/ |
| Swagger API 文件 | http://localhost:8095/swagger-ui.html |
| H2 資料庫主控台 | http://localhost:8095/h2-console |

H2 主控台登入資訊：
- JDBC URL：`jdbc:h2:file:./data/bomdb`
- User Name：`sa`
- Password：（留空）

> 資料庫每次啟動都會依 `schema.sql`／`data.sql` 重新建立並帶入示範資料，不會保留上次啟動後的異動內容。

## 基本操作說明

前端頁面（http://localhost:8095/）由上到下分為六個區塊，建議依以下順序操作：

### 1. 物料主檔管理
新增物料時填入編碼、名稱、單位、單價（若是組裝件本身沒有直接單價，可留空，成本會由底下子物料加總算出）。所有輸入物料編碼的地方，之後都會以此處建立的資料作為搜尋式下拉選單的來源。

### 2. BOM 結構維護
選擇「父物料」「子物料」並輸入數量，代表「父物料的 BOM 裡包含這顆子物料多少數量」。若要建立一個全新的 BOM（例如新產品），先在物料主檔新增該產品的物料，再回到這裡把它當作父物料，加入第一筆組成即可，不需要額外的「建立 BOM」步驟。

### 3. BOM 查詢
在根節點物料欄位輸入關鍵字，選擇要查詢的產品，按下「查詢結構」可看到完整展開的樹狀圖；按下「計算成本」則列出總成本與每一項物料的成本明細。

### 4. 替代方案管理
新增一個替代方案（給它一個 Key 與名稱），用來將一組替代規則命名分組（例如某次缺料時的因應策略）。

### 5. 方案明細維護
選擇要編輯的方案後，設定「主料 → 替代料」的規則：替代比例（預設 1，代表 1:1 替換）與替代料單價。這裡不會輸入數量，數量是查詢／算成本時才決定。

### 6. 套用替代方案查詢／計算成本
回到區塊一「BOM 查詢」，勾選要套用的替代方案，畫面會列出該方案影響到的主料，針對每一筆輸入這次要替換的數量後，再按「查詢結構」或「計算成本」，即可看到套用替代後的樹狀圖與成本變化。

### 所有方案明細查詢
不需要先選定方案，可直接用關鍵字搜尋所有方案裡「誰可以換誰」的規則，方便快速確認某顆物料目前有哪些替代選項。

## API 文件

完整 API 規格與線上測試介面請參考 Swagger UI：http://localhost:8095/swagger-ui.html

主要資源：

| 資源 | 路徑 |
|---|---|
| 物料主檔 | `/api/materials` |
| BOM 組成關係 | `/api/bom-components` |
| BOM 查詢／成本計算 | `/api/bom/{rootCode}/structure`、`/api/bom/{rootCode}/cost`、`/api/bom/roots` |
| 替代方案 | `/api/scenarios` |
| 方案明細 | `/api/scenarios/{scenarioKey}/items`、`/api/scenarios/items` |
